April 2025 | Kafka and Microservices ....

In preparation for my new job, I decided to learn the technologies I'd be using and build a useful application with Kafka message queues and a microservice architecture.


The microservices were:

  1. One container per coin to track the price every 5 seconds and pushes the new price to a Kafka message queue
  2. One container to consume each Kafka message and updates 2 MongoDB collections with current and historical price data
  3. An nginx container to serve static front-end assets, handwritten HTML because React was unnecessary
  4. One stateless, scalable, RESTful API container to handle front end requests


The application has a some features:

  1. View crypto prices tracked from multiple sources, as some coins cannot be tracked on all APIs
  2. Graph important trading periods against crypto price, with details of expected movements
  3. Track trades and view profit/loss for each asset


Somethings could definitely be improved before this application would be production ready - logging, monitoring, k8s, an authentication microservice, a microservice to send an alert when a coin reaches a certain price, loading historical data, etc

July 2022 | Australia Post parcel tracking dashboard

This product tracks Australia Post parcels using a React frontend and Typescript API backed by PostgreSQL.


This project was an exercise in implementing an Object Relational Mapper (ORM) like Java's Hibernate in Typescript. The initial scope included reimplementing Hibernate's annotation based configuration to automatically generate JOIN statements in SQL queries.


The back end parcel tracking system uses a message queue for adding new parcels with a background cron service to provide regular synchronizations with the Auspost API for packge updates.


A notification script written in golang alerts the user whenever the package location changes.

2017 | Snooping SSL proxy that injects JS and steals credentials

In this project I configured a HTTP proxy to generate bogus SSL certificates and inject JavaScript into any page visited by the proxy user. The injected JavaScript would record all form data that was entered by the proxy user, such as username and passwords. A custom NodeJS Command and Control server was created to record all information entered on the web page.


This project was inspired by a DEFCON 20 talk. Using open source software, a malicious HTTP proxy intercepts SSL connections by generating SSL certificates on the fly to inject Javascript into every website the user visits, stealing every entered username and password combo.


This demonstrates how off the shelf software can be used for intercepting SSL traffic and highjacking network traffic, but it is limited in effect by not being able to generate real certificates on the fly. This is not necessarily a problem for a well resourced attacker.


A modern solution to this problem is to implement mitmproxy and a custom C&C server.
